# Release Checklist

## Purpose
Standard checklist for releases.

## Sections
1. Pre-Release (tasks done, CI passing, CHANGELOG, version updated)
2. Quality Gate (tests, coverage, static analysis, security scan)
3. Documentation (README, API docs, migration guide)
4. Release Execution (branch, tag, build, deploy, merge)
5. Post-Release (announce, update ROADMAP, increment version, monitor)

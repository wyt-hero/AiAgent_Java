# Review Checklist

## Purpose
Standard checklist for code reviews.

## Sections
1. AIOS Compliance (architecture, rules, boundaries)
2. Code Quality (no logic in controllers, Javadoc, tests, no hardcoded values)
3. Security (no secrets, input validation, no SQL injection)
4. Documentation (CHANGELOG, AIOS updates, OpenAPI)
5. Testing (unit, integration, coverage, edge cases)
6. Performance (no N+1, indexed queries, no blocking calls)
